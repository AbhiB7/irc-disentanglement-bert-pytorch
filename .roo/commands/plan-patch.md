Before making any code changes:

1. Read the relevant files first.
2. Summarize the current behavior and architecture in plain English.
3. Identify what looks incomplete, incorrect, or risky.
4. Propose the smallest safe patch that addresses the problem.
5. List which files should be changed and why.
6. Do not write code yet unless I explicitly approve the plan.
7. Keep the explanation concise and easy to understand.
